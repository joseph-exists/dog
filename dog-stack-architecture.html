Dog Stack Architecture

A local-first collaboration stack for agents, rooms, stories, repos, projects, and Kennel-backed workspaces. The diagram separates user-facing product surfaces, backend service groups, and Docker/runtime infrastructure.

docker-compose.yml: 10 app services
Traefik optional: public routing
Redis: realtime + worker channels
Gittin/Gogs: user + shadow repos

Read The Diagram

  • Product surfaces at the top call the FastAPI backend through generated client code.
  • Backend service groups show the major code boundaries, not every module.
  • Infra boxes show the Docker services and external runtimes that make the product work.

Realtime Path

  • Room writes go through event_emitter and append immutable events.
  • Redis fans out persisted events and ephemeral token deltas to each worker.
  • WebSocket reconnects replay missed room sequences from the database.

Durable Runtime Context

  • Stories provide shared room state machines and prompt-visible runtime projection.
  • Shadow snapshots version rooms, stories, agents, personas, models, and providers.
  • Repos and workspaces feed code context and executable environments into rooms.

Operational Notes

  • Kennel is privileged LXC infrastructure intended for local or owned deployments.
  • Gittin/Gogs is hidden behind Dog authorization for user repos and Shadow repos.
  • Traefik is optional for local direct-port development, but important for routed deployments.
